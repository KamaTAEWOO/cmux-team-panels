# Team Panels — cmux 팀 에이전트 스킬 (격리 워크스페이스 + 조건부 체인)

cmux에서 실행하면 **새 워크스페이스**가 생성되고 그 안의 기획 패널이 작업을 시작합니다.
기획 단계가 팀 리드로서 후속 역할 필요 여부를 판단하고, 모든 역할이 완료되면
워크스페이스 전체가 일괄 닫힙니다. 현재 작업 중인 워크스페이스에는 전혀 영향이 없습니다.

## 동작 흐름

```
[새 워크스페이스 생성] → [기획] 실행 → 후속 역할 필요성 판단 (<next-roles> 마커)
    ├─ 불필요: 기획이 직접 완결 → 5초 후 워크스페이스 close
    └─ 필요:   지정된 역할만 순차 체인 → 마지막 완료 후 워크스페이스 close
```

- **격리 워크스페이스**: 체인은 새 워크스페이스(`team-panels`) 안에서만 진행 — 기존 작업 공간 오염 없음
- **기획 = 팀 리드**: 간단한 작업은 자기가 다 처리(파일 저장 포함)
- 기획 산출물 마지막에 `<next-roles>02,03,04,05</next-roles>` 마커로 후속 역할 지정
  - 빈 태그(`<next-roles></next-roles>`)면 여기서 종료
  - 번호: 02=디자인 03=개발 04=코드리뷰 05=테스트 (필요한 것만 선택)
- 중간 역할은 완료 후에도 패널 유지 — 산출물 확인 가능
- 마지막 역할 완료 후 **워크스페이스 전체가 close** (모든 패널 일괄 정리)
- 각 역할은 이전 단계 산출물을 프롬프트 컨텍스트로 전달받음
- 각 패널의 `claude -p`는 `--dangerously-skip-permissions`로 실행 (파일 쓰기 권한 자동 허용)

## 역할 풀 (기획이 필요한 것만 골라 호출)

| 번호 | 역할 | 산출물 |
|---|---|---|
| 01 | 기획(PM/리드) | 요구사항, 유저 스토리, 수용 기준, 후속 역할 선택 |
| 02 | 리서치 | 시장/경쟁사/유저 인사이트, 참고 사례 |
| 03 | 아키텍트 | 시스템 다이어그램, 데이터 흐름, 기술 스택 근거 |
| 04 | DBA | 엔티티·스키마·인덱스, 마이그레이션 전략 |
| 05 | 디자인 | 화면 흐름, 컴포넌트, 인터랙션, 와이어프레임 |
| 06 | UX라이터 | 카피, 에러/성공/빈상태 문구, 톤앤매너 |
| 07 | 개발 | 파일 구조, 함수 시그니처, 의사코드, 에러 처리 |
| 08 | 보안 | STRIDE 위협, OWASP 체크, 인증·인가, 민감 데이터 |
| 09 | 코드리뷰 | 리스크, 개선 체크리스트, 추천 리팩토링 |
| 10 | DevOps/SRE | CI/CD, 환경 구성, 모니터링, 롤백 전략 |
| 11 | 테스트(QA) | 테스트 피라미드, 케이스, 자동화 범위 |
| 12 | 테크라이터 | README, API 명세, 온보딩 가이드 |

**호출 예시** (기획이 작업 종류에 따라 `<next-roles>` 조합)
- 단순 카피 수정: `<next-roles>06</next-roles>`
- 일반 웹 기능: `<next-roles>02,03,05,07,09,11</next-roles>`
- 보안 민감 기능: `<next-roles>02,03,04,08,07,09,11</next-roles>`
- 릴리즈 준비: `<next-roles>09,11,10,12</next-roles>`
- 간단 답변: `<next-roles></next-roles>` (기획이 직접 완결)

## 실행 방법

사용자가 이 스킬을 호출하면:

1. **인자 확인**: 사용자 요청에서 작업 설명 추출 (없으면 물어봄)
2. **환경 확인**: `$CMUX_WORKSPACE_ID` 존재 확인
3. **실행**:
   ```bash
   bash ~/.claude/skills/team-panels/run.sh "작업 설명"
   ```
4. **결과 안내**: 워크디렉토리 경로 (`~/.cmux-team/<timestamp>/`) 알림
5. **완료 대기하지 않음**: `run.sh`는 첫 패널만 띄우고 즉시 반환.
   체인은 각 패널이 자율적으로 진행

## 산출물 위치

`~/.cmux-team/<timestamp>/outputs/` 에 `01-기획.md`, `02-디자인.md` 등으로 저장

## 패널 이름 표시

- `cmux rename-tab` 으로 탭 이름 설정
- 터미널 escape sequence (`\033]0;NAME\007`)로 타이틀 동기화
- 둘 중 하나라도 동작하면 사이드바에 역할명 표시됨

## 주의

- cmux 터미널 세션 안에서만 작동 (`CMUX_WORKSPACE_ID` 필요)
- 각 패널은 별도 `claude -p` 프로세스 → 현재 대화 컨텍스트와 독립
- 완료 시 패널은 닫히지만 산출물 파일(.md)은 워크디렉토리에 남음
- 체인 중간에 실패해도 다음 패널 열기는 시도됨 (부분 실패 내성)
